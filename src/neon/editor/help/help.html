Getting started

Create a new module

Click on File > New... to create a new module. This will open a dialog box requesting an id for the new module. This id should be unique. If the id already exists, the module will not be created.

The new module will be created without any content. The default name of a module is the same as the module id.

Open an existing module

Click on File > Open... to open a module. Select the folder that contains the module and click on the Select Folder button.

The module folder should be placed in the neon data folder before opening. If the folder is located in another location, or does not actually contain a module, the editor will behave in exciting and unexpected ways.

Save a module

After loading or creating a module, the Save menu item will become available in the File menu.

The editor keeps a backup in the neon temp folder. In case of emergency, unsaved data can be manually recovered from here. Do not start the editor or a game, however, as that will overwrite this folder.

Resources

All editable data in a module is represented by a resource. They are shown in different colors and styles:
  • default: a resource from the currently active module that was not present in any parent module.
  • gray: a resource from a parent module that was not redefined in the currently active module.
  • dark red: a resource from a parent module that was redefined in the currently active module.
  • italic: a resource that was edited, but not yet saved.
